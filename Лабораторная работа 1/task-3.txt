Контрольные вопросы по Лабораторной работе №1
Вопрос 1. Что такое HTTP и какие его основные методы вы знаете? Для чего используется каждый из них?

Ответ: HTTP (Hypertext Transfer Protocol) — это протокол передачи гипертекста, основной стандарт обмена данными в интернете. Он работает по модели «клиент-сервер». Основные методы:

GET: Используется для запроса и получения данных с сервера (например, просмотр страницы). Не должен изменять состояние данных на сервере.

POST: Используется для отправки данных на сервер (например, регистрация или загрузка файла). Создает новый ресурс или обрабатывает данные.

PUT: Используется для полного обновления существующего ресурса.

DELETE: Используется для удаления указанного ресурса.

Вопрос 2. Объясните назначение библиотеки requests в Python. В чем ее преимущества перед встроенными модулями для работы с HTTP?

Ответ: Библиотека requests предназначена для отправки HTTP-запросов и обработки ответов.

Преимущества: Она обладает «человекочитаемым» интерфейсом в отличие от стандартной библиотеки urllib. Она автоматически обрабатывает кодировки, поддерживает сессии, легко работает с cookies и позволяет извлекать JSON-данные одной командой .json(), что значительно сокращает объем кода.

Вопрос 3. Какие коды состояния HTTP указывают на успешное выполнение запроса?

Ответ: На успех указывают коды группы 2xx. Самый распространенный — 200 OK, который означает, что запрос был успешно обработан и сервер вернул ожидаемые данные.

Вопрос 4. Для чего используются HTTP-заголовки? Приведите примеры заголовков, которые могут быть важны при работе с веб-сервисами.

HTTP-заголовки передают метаинформацию о запросе или ответе: тип данных, кодировку, авторизацию, информацию о клиенте.

Важные заголовки:

User-Agent — идентифицирует клиент (браузер/приложение)

Authorization — передаёт токен или учётные данные для аутентификации

Content-Type — указывает формат данных (application/json, text/html)

Accept — сообщает серверу, какой формат ответа ожидает клиент

Cookie — передаёт сохранённые данные сессии

Вопрос 5.  Что такое requests.Session() и когда целесообразно его использовать?

requests.Session() — объект, который сохраняет параметры между запросами: cookies, заголовки, настройки аутентификации.

Когда использовать:

При нескольких запросах к одному хосту

Когда нужно сохранять cookies (например, после входа на сайт)

Для повторного использования одинаковых заголовков во всех запросах.

Вопрос 6. Объясните назначение библиотеки BeautifulSoup. Для чего она нужна?

BeautifulSoup — библиотека для парсинга HTML и XML. Она преобразует HTML-код в структурированное дерево, по которому удобно искать, извлекать и навигировать по элементам страницы.

Вопрос 7. Какие парсеры можно использовать с BeautifulSoup? Какой из них рекомендуется и почему?

html.parser — встроенный, не требует установки

lxml — быстрый, гибкий, требует установки

html5lib — самый медленный, но максимально приближен к браузеру

Рекомендуется lxml, потому что он быстрее остальных и поддерживает все возможности навигации.

Вопрос 8. Что такое CSS-селекторы и как их использовать для поиска элементов в BeautifulSoup?

CSS-селекторы — это строки, описывающие элементы по тегам, классам, id и их вложенности.

Использование:

soup.select('selector') — возвращает список элементов

soup.select_one('selector') — возвращает первый элемент

Примеры:

soup.select('a') — все ссылки

soup.select('.news-title') — все элементы с классом news-title

soup.select('#header') — элемент с id header

soup.select('div.content a') — ссылки внутри div с классом content

Вопрос 9. Как извлечь текстовое содержимое элемента и значение его атрибута после того, как элемент найден?

Ответ: В библиотеке BeautifulSoup для этого используются следующие способы:

Текст: Свойство .text или метод .get_text(). Например: element.text.strip().

Атрибут: Метод .get('название_атрибута') или обращение по ключу, как в словаре. Например: element.get('href') для получения ссылки.

Вопрос 10. Что такое JSON? Опишите его основные структуры данных.

Ответ: JSON (JavaScript Object Notation) — это легкий текстовый формат обмена данными, основанный на языке JavaScript, но независимый от него. Основные структуры:

Объект (Object): Неупорядоченный набор пар «ключ: значение», заключенный в фигурные скобки {}. В Python соответствует словарю (dict).

Массив (Array): Упорядоченный список значений, заключенный в квадратные скобки []. В Python соответствует списку (list).

Вопрос 11. Какие потенциальные проблемы могут возникнуть при работе с внешними API и как их следует обрабатывать?

Ответ: Лимиты запросов (Rate Limiting): Ограничение на количество запросов в минуту/час. Решается использованием пауз (time.sleep()) или кэшированием данных.

Сетевые ошибки: Отсутствие интернета или падение сервера. Решается использованием блоков try-except и проверкой статус-кодов.

Изменение структуры данных: API может обновиться, и ключи в JSON изменятся. Решается проверкой наличия ключей перед использованием.

Ошибки авторизации: Истечение срока действия API-ключа.

Вопрос 12. Почему важно проверять robots.txt перед парсингом сайта? Какие еще этические и технические аспекты следует учитывать при веб-скрейпинге?

Ответ: Robots.txt: Это файл, в котором владелец сайта указывает, какие разделы запрещено посещать автоматическим скриптам. Игнорирование этого файла — нарушение сетевого этикета и правил сайта.

Этические аспекты: Нельзя создавать слишком высокую нагрузку на сервер (частые запросы могут парализовать работу сайта).

Технические аспекты: Использование актуального User-Agent, чтобы сервер понимал, кто делает запрос, и соблюдение авторских прав на скачиваемый контент.
